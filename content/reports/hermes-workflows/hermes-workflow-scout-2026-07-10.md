# Hermes workflow items worth opening today

- 🔥 Best: Instar is a close Hermes-adjacent reference for persistent Claude Code agents with scheduling, sessions, memory, and Telegram access. https://github.com/JKHeadley/instar
- DorkOS is a self-hosted mission-control dashboard for Claude Code, Codex, and OpenCode sessions with approvals and scheduling. https://github.com/dork-labs/dorkos
- ClawMetry offers real-time observability across Claude Code, Codex, OpenClaw, and other agent runtimes, useful as a model for Hermes agent telemetry. https://github.com/vivekchand/clawmetry
- AgentHub is a self-hosted control plane for long-lived coding agents with ACP timelines, team workflows, and remote execution nodes. https://github.com/hawkingrei/agenthub
- Wick packages Claude, Codex, or Gemini as Slack, Telegram, and web agents, making it a practical gateway-hosting pattern to compare against Hermes. https://github.com/yogasw/wick
- Agent Reachout shows a narrow human-in-the-loop Telegram bridge where Claude Code asks for decisions or reports completion instead of relying on dashboards. https://github.com/vibe-with-me-tools/agent-reachout
- Dagu is a local-first workflow engine with Web UI, MCP support, and coding-agent harness hooks that could complement proactive Hermes cron workflows. https://github.com/dagucloud/dagu
- Droidctx generates Markdown infrastructure context from production systems so coding agents start with deploy/runtime facts instead of stale guesses. https://github.com/DrDroidLab/context-builder
- Routine Scheduler explores git-backed scheduled LLM routines with a web UI for watching or intervening in running conversations. https://github.com/wehnsdaefflae/routine-scheduler
- Oh-My-OpenClaw ports chat-native OpenClaw orchestration patterns to Discord and Telegram, useful for comparing topic-scoped agent UX. https://github.com/happycastle114/oh-my-openclaw
