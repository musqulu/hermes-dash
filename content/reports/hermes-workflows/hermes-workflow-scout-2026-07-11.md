# Hermes workflow items worth opening today

- 🔥 Best: Codelight is a compact reference for agent-neutral mobile and desktop status, remote permission approvals, question answering, usage display, and conversation following across Claude Code, Codex, and Copilot. https://github.com/henrikekblad/codelight
- Lucarne shows a zero-intrusion Telegram/WeChat bridge for local coding agents where approvals, clarifying questions, and failure notifications become actionable mobile events without hooks or MCP. https://github.com/tuchg/Lucarne
- Codex Mate is a local-first control plane for switching providers, managing sessions, and orchestrating Claude Code, Codex, OpenCode, and OpenClaw from one dashboard. https://github.com/SakuraByteCore/codexmate
- OPEX is a Rust self-hosted agent gateway with multi-platform chat frontends, memory, cron, secrets, approvals, and per-agent heartbeats that overlaps strongly with Hermes gateway infrastructure. https://github.com/AronMav/opex
- Talon combines Telegram, Discord, Teams, terminal, Flutter companion apps, MCP tools, proactive goals, and memory consolidation into a broad multi-frontend agent harness. https://github.com/dylanneve1/talon
- Agentic Sandbox provides a self-hostable runtime for persistent autonomous coding agents with KVM or container isolation, management APIs, a web dashboard, and Claude/Codex/OpenCode images. https://github.com/jmagly/agentic-sandbox
- Projmux is a tmux-native multi-agent workspace for Claude Code, Codex, and Antigravity with hook-driven attention and agent-aware session resume. https://github.com/crevissepartners/projmux
- Wmux is a Windows-native agent multiplexer that fans prompts into isolated worktree agents, survives reboot, and adds browser automation, identity, and human approval gates. https://github.com/openwong2kim/wmux
- Tmux Agents Inbox is a small but sharp UX pattern for seeing which Claude Code sessions are waiting, done, or working across tmux and jumping straight to the next one. https://github.com/ctretyak/tmux-agents-inbox
- Kastra is worth studying as a policy-enforcement layer that checks Claude Code, Codex CLI, Cursor, shell commands, file edits, and API calls before execution rather than only logging them afterward. https://kastra.ai/
