# Hermes/agent workflows worth opening today

- 🔥 Best: Hermes Evolution is a self-improving Hermes Agent fork that researches improvements, proposes updates, and turns daily usage into a learning loop you could adapt for personal-operator maintenance. Source: https://github.com/Lexus2016/hermes-agent-evolution
- tagteam runs multiple headless coding agents such as Codex and Claude as one inspectable CLI workflow with saved briefs, diffs, reviews, and test results. Source: https://github.com/cephalopod-ai/tagteam
- Crewplane turns Markdown DAGs into repeatable agent workflows for Claude Code, Codex, Gemini, Copilot, or any CLI, with resume/skip boundaries and on-disk evidence. Source: https://github.com/crewplaneai/crewplane
- Circus Chief is a touch-friendly control plane for Claude Code, OpenAI Codex, and Gemini agents, including Kanban-driven prompts, scheduled retries, and phone-friendly supervision. Source: https://github.com/ferrislucas/Circus-Chief
- aharness makes Codex-style long-running agent work executable as gated state machines so approvals, recovery rules, and evidence checks are enforced instead of left to prompts. Source: https://github.com/Alfredvc/aharness
- telegram-mcp exposes Telegram chats, groups, messages, media, contacts, and settings to MCP-compatible clients, useful for building Telegram-based personal assistant workflows. Source: https://github.com/chigwell/telegram-mcp
- Moltis is a Rust personal agent server with sandboxed execution, memory, voice, Telegram/WhatsApp/Discord/Teams channels, and MCP tools for a self-hosted operator setup. Source: https://github.com/moltis-org/moltis
- Marketing Analytics MCP gives an LLM read-only access to GSC, GA4, Google Ads, HubSpot, and Bing so you can ask natural-language growth and SEO questions across dashboards. Source: https://github.com/Dweeb1578/marketing-analytics-mcp
- Hiroshi OS is a local assistant daemon with Telegram/Discord/Matrix gateways, file and command control, vector search, MCP servers, cron tasks, and sandbox execution. Source: https://github.com/hiroshi-os/Hiroshi
- peerd is a browser-native agent harness for running autonomous workflows inside Chrome or Firefox, worth watching for research, monitoring, and web-operator experiments. Source: https://github.com/NotASithLord/peerd
