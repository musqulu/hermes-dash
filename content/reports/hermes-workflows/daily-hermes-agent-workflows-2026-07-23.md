# Hermes/agent workflows worth opening today

- 🔥 Best: Ductor is a Telegram control plane for Claude Code, Codex CLI, and Gemini CLI with live streaming, persistent memory, cron jobs, webhooks, and Docker sandboxing. https://github.com/PleasePrompto/ductor
- ccteam turns Claude Code, Codex, Grok, Kimi, and other coding agents into a cross-machine team you can steer from Telegram, Lark, or a browser tab. https://github.com/firstintent/ccteam
- Hermes Agent’s official Team Telegram Assistant guide is the cleanest reference for turning a Telegram group into a routed personal or team operator. https://hermes-agent.nousresearch.com/docs/guides/team-telegram-assistant
- Hermes Agent’s optional Codex app-server runtime lets Hermes wrap Codex for terminal commands, file edits, sandboxing, and MCP tool calls while keeping Hermes sessions, memory, skills, and slash commands. https://hermes-agent.nousresearch.com/docs/user-guide/features/codex-app-server-runtime
- free-hermes shows a practical Fly.io deployment pattern for running Hermes Agent cheaply while waking on Telegram webhooks or scheduled HTTP pokes. https://github.com/zlen2024/free-hermes
- hermes-mobile is an open-source mobile client for Hermes Agent with real-time chat, session resume, and cron management from a phone. https://github.com/Hy4ri/hermes-mobile
- Fluxito is a self-hostable MCP marketing analytics workspace that lets an agent plan, implement, audit, and report across many marketing and analytics platforms. https://github.com/digitalXperiments/fluxito
- send16-mcp gives MCP agents 79 tools for transactional and marketing email, including a zero-DNS sandbox sender for fast campaign and notification experiments. https://github.com/spruikco/send16-mcp
- awesome-n8n-templates collects 280+ ready-to-import automations for Telegram, OpenAI, Gmail, Sheets, Notion, WhatsApp, social media, RAG, and document processing. https://github.com/enescingoz/awesome-n8n-templates
- Octopus LinkedIn is a governed LinkedIn MCP loop for drafting, approving, publishing, commenting, and analyzing via the official LinkedIn API. https://github.com/octoryn/octopus-linkedin
