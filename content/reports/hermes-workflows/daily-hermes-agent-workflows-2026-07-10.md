# Hermes/agent workflows worth opening today

- 🔥 Best: Talon is a fresh multi-platform agent harness that runs on Telegram, Discord, Teams, terminal, and mobile while routing to Codex, Claude, OpenAI Agents, and MCP tools. https://github.com/dylanneve1/talon
- DorkOS is a self-hosted mission-control cockpit for running Claude Code, Codex, and OpenCode sessions with approvals, scheduling, and messaging. https://github.com/dork-labs/dorkos
- Instar packages persistent Claude Code agents with scheduling, sessions, memory, and Telegram, which maps closely to a personal operator setup. https://github.com/JKHeadley/instar
- Billy is a self-hosted Telegram AI assistant that uses specialist sub-agents for email, CRM, web research, documents, images, and reusable skills. https://github.com/surfgolfski-dev/billy
- AI Agent Skills is a portable skill library for Claude Code, Cursor, Codex CLI, Gemini CLI, Goose, and other agent clients, useful for standardizing repeatable workflows. https://github.com/sanjay3290/ai-skills
- aharness turns long Codex-style workflows into executable state machines with enforced gates instead of relying on a prompt to remember the process. https://github.com/Alfredvc/aharness
- CodeAlmanac builds a local self-updating codebase wiki so coding agents can understand project structure and decisions before editing. https://github.com/AlmanacCode/codealmanac
- Live Log Viewer gives a local dashboard for Claude Code and Codex sessions, subagents, shell tasks, and live tool output across `~/.claude` and `~/.codex`. https://github.com/Latand/live-log-viewer-next
- Mnemosyne is a zero-dependency SQLite-backed memory layer with MCP support that could give Hermes-style agents fast durable recall. https://github.com/mnemosyne-oss/mnemosyne
- This GitHub code-review-agent walkthrough is a practical template for turning repo events into automated review comments and can be adapted for product QA loops. https://debugger.substack.com/p/how-to-build-github-code-review-agent
