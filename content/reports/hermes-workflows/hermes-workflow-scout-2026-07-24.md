# Hermes workflow items worth opening today

- 🔥 Best: aharness turns fragile “follow this process” prompts into executable state-machine gates for long-running Codex-style agent workflows. https://github.com/Alfredvc/aharness
- OtoDock is a self-hosted server for running Claude Code and Codex as a coordinated team of coding agents, useful inspiration for persistent multi-agent infrastructure. https://github.com/OtoDock/oto-dock
- OneCLI is an OSS credential gateway that stores secrets once and injects them into tools without exposing keys directly to AI agents. https://github.com/onecli/onecli
- mwe-mcp offers a self-hosted MCP memory wiki with multi-agent, multi-user, time-aware fragments that maps well to Hermes memory rollups. https://github.com/Fr4nZ82/mwe-mcp
- Kompass provides token-efficient, composable workflow steering for coding agents, a practical pattern for topic-scoped agent runbooks. https://github.com/kompassdev/kompass
- AppIdeasAutomated shows a simple daily Claude Code routine that picks one task, builds it, sanity-checks it, updates progress, commits, and pushes. https://github.com/Ventixz/AppIdeasAutomated
- hermes-agentic-ai-infrastructure documents a VPS-based autonomous agent setup with 45 cron workflows, centralized LLM routing, observability, and knowledge management. https://github.com/irfanzulkifle/hermes-agentic-ai-infrastructure
- Setoku is a self-hosted knowledge server for AI agents, worth checking as a possible external memory/context store pattern. https://setoku.com/
- Screenpipe records work activity and turns it into agents, relevant for proactive workflow capture and agent-assist context generation. https://news.ycombinator.com/item?id=49024620
