# Hermes workflow items worth opening today

- 🔥 Best: `agmsg` is a Bash + SQLite messaging layer for letting Claude Code, Codex, Gemini, Copilot, and similar CLI agents coordinate as one team: https://github.com/fujibee/agmsg
- `context-infrastructure` packages persistent memory, personal rules, skills, and scheduled observations as a local context layer for coding agents: https://github.com/grapeot/context-infrastructure
- `memorix` is an MCP-based cross-agent memory layer that explicitly lists Hermes Agent, Claude Code, Codex, Cursor, and Gemini CLI compatibility: https://github.com/AVIDS2/memorix
- `tokentelemetry` is a local dashboard for tracking tokens, sessions, tool calls, and reasoning across Hermes Agent and other autonomous coding agents: https://github.com/VasiHemanth/tokentelemetry
- `dagu` is a mature local-first YAML workflow engine with a Web UI that can be adapted as a cron/proactive-agent orchestration layer: https://github.com/dagucloud/dagu
- `agentic-sandbox` is a self-hostable runtime for persistent autonomous coding agents with KVM/rootless isolation, mission dispatch, shared storage, and a web dashboard: https://github.com/jmagly/agentic-sandbox
- `handoff-mcp` stores prioritized cross-session agent briefs as plain Markdown with temporal supersession and optional multi-device git sync: https://github.com/kirill-sviridov/handoff-mcp
- `Vault-Agent-Memory` focuses on reviewable, auditable, local-first shared memory for agents using SQLite and MCP: https://github.com/zycaskevin/Vault-Agent-Memory
- `mcp-local-hub` proposes one shared workstation MCP daemon instead of each AI client spawning redundant stdio servers: https://github.com/applicate2628/mcp-local-hub
- `macmini-control-room` is a prompt-as-Markdown, cron-driven web control plane for autonomous Claude Code agents on a Mac Mini: https://github.com/nicolasmertens/macmini-control-room
