# Hermes/agent workflows worth opening today

- 🔥 Best: Hermes Atlas is a live directory of Hermes Agent skills, plugins, deployment templates, forks, and integrations worth mining before building custom workflows. https://github.com/ksimback/hermes-ecosystem
- Railway Hermes OAuth Image packages Hermes Agent with OpenAI Codex, Nous subscription delegation, messaging gateways, persistent Railway storage, and a health-only public surface. https://github.com/qf1lzx/railway-hermes-oauth-image
- ultracodex lets Claude-style workflow scripts fan out work to OpenAI Codex agents while keeping the stronger model for planning and verification. https://github.com/YuanpingSong/ultracodex
- relaymux is a tmux-backed harness for launching Codex, Claude, or Pi agents from Telegram/iMessage while keeping visible local terminal sessions. https://github.com/mupt-ai/relaymux
- Antigravity Telegram Suite shows a polished phone-control pattern for AI agents with model switching, workspace control, screenshots, files, and multi-agent workflows. https://github.com/emreturkmencom/antigravity-telegram-suite
- Loomem is a local-first MCP memory layer that gives Codex, Claude, ChatGPT, or other MCP clients shared facts, decisions, history, and hybrid search. https://github.com/vvooki-sys/loomem
- Robin is a Claude Code-driven personal assistant daemon with MCP servers, session capture, Gmail/Calendar/Linear integrations, scheduling, and long-term SQLite memory. https://github.com/kevinkiklee/robin-assistant
- Gypsum is a self-hosted Markdown wiki with built-in MCP tools so an agent can create, edit, search, and maintain a second brain over time. https://github.com/mnorrsken/gypsum
- Burrow exposes Kubernetes self-hosting operations over MCP so coding agents can deploy, debug, scale, and manage apps behind human approval guardrails. https://github.com/burrow-cloud/burrow
- content-engine is an agent-owned Instagram Reels pipeline that researches topics, writes storyboards, renders Remotion videos, sends Telegram approvals, posts, and learns from performance. https://github.com/noahl25/content-engine
