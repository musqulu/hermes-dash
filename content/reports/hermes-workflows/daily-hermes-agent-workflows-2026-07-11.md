# Hermes/agent workflows worth opening today

- 🔥 Best: Hermes’ Team Telegram Assistant guide is the most directly reusable blueprint for turning a Hermes setup into a phone-controlled operator with team access and operational guardrails. https://hermes-agent.nousresearch.com/docs/guides/team-telegram-assistant
- Hermes’ bundled Codex skill docs are worth wiring into your setup so Hermes can delegate coding, reviews, and implementation work to OpenAI Codex from inside the agent workflow. https://hermes-agent.nousresearch.com/docs/user-guide/skills/bundled/autonomous-ai-agents/autonomous-ai-agents-codex
- Kai is a local Telegram-first engineering assistant with repo-aware coding, memory, scheduling, PR review, files, and persistent agent sessions on your own machine. https://github.com/dcellison/kai
- CatClaw is a Rust daemon that turns Claude Code into a multi-agent, multi-channel assistant across Telegram, Discord, Slack, and TUI with SQLite persistence and tool approvals. https://github.com/CatiesGames/catclaw
- Caliper gives Hermes, Codex, and Claude Code skills a practical pass@k reliability harness so you can test whether an agent workflow actually works before relying on it. https://github.com/edonadei/caliper
- 9lives is a self-healing Playwright test runner that can call Claude Code, Codex, or OpenCode to repair broken tests while always showing a diff. https://github.com/Quality-Max/9lives
- Squish is a local-first MCP memory runtime that connects Claude Code, Codex, ChatGPT, and other agents to persistent sources, connectors, and knowledge graphs. https://github.com/michielhdoteth/squish
- codex-in-claude exposes the Codex CLI as a FastMCP plugin for Claude Code so one agent can delegate implementation, review, and second-opinion tasks to another model. https://github.com/briandconnelly/codex-in-claude
- Growth OS is an agent-ready marketing system that points at a business URL, saves ICP/USP context, audits SEO, speed, GEO, competitors, GA4, and Search Console, then delegates growth tasks. https://github.com/nocodework/growth-os
- Mnemosyne is a Hermes-first SQLite memory layer for agents that need fast, local, zero-dependency durable memory across Codex, Claude Code, Cursor, and custom workflows. https://github.com/mnemosyne-oss/mnemosyne
